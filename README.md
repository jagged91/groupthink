# Groupthink
### A governance framework for groups of any size and purpose - Version 0.6

---

## What is this?

Groupthink is a framework for how groups of people can govern themselves - make decisions together, manage shared resources, relate to other groups, and protect what matters. It is designed to work at any scale, from a gaming group to a territorial federation.

The name is a deliberate reclamation. "Groupthink" originally described the pathology of conformity - groups suppressing dissent to maintain harmony. This framework is the opposite: a structure where dissent is protected, departure is unconditional, and consent is ongoing.

## How it works

The framework asks one question: **What does your group do?**

Every Groupthink group starts with the **Core** - values, rights, and basic mechanics that apply always. Beyond that, the framework provides guidance for specific activities. You use what you need.

## Structure

```
Declaration Zero.md        - Why this framework exists

0.Documents/
  Core/                    - What every Groupthink group shares
    0.Values.md            - Floors (non-negotiable) and Directions (aspirational)
    1.Rights.md            - Rights of persons within groups
    2.Implementation.md    - Basic group mechanics: records, consent, splits

  Stewardship/             - When a group manages shared resources
    0.Values.md            - The commons, sustainability, responsibility
    1.Rights.md            - Stakeholder standing, the right to invoke
    2.Implementation.md    - Identifying commons, invocation process, roles

  Federation/              - When groups relate to other groups
    0.Values.md            - Sovereignty, voluntary association, bounded scope
    1.Rights.md            - Rights of groups in relation to each other
    2.Implementation.md    - Federations, compacts, iterative deliberation

  Provision/               - When a group supplies essential needs
    0.Values.md            - Structural dependency, genuine exit, alternatives
    1.Rights.md            - Rights of members who depend on their group
    2.Implementation.md    - Ensuring exit is real, structural health checks

  Defense/                 - When irreversible harm must be prevented
    0.Values.md            - Irreversibility as threshold, defense not police
    1.Rights.md            - Due process, proportionality, community control
    2.Implementation.md    - Activation, authorization, conduct, review
```

## Quick start

- **Any group**: Read `Documents/Core/0.Values.md`, `Documents/Core/1.Rights.md`, `Documents/Core/2.Implementation.md`.
- **Managing shared resources**: Add `Documents/Stewardship/`.
- **Relating to other groups**: Add `Documents/Federation/`.
- **Providing essentials to members**: Add `Documents/Provision/`.
- **Facing irreversible threats**: Add `Documents/Defense/`.

## Principles

- **Floors and Directions.** Some values are absolute lines that cannot be crossed (no one prevented from leaving, no invisible decisions, no irreversible harm). Others are directions to move toward continuously (maximum consent, minimum coercion, maximum transparency).
- **Values before method.** The framework says what must be honoured. How you honour it is up to your group.
- **Use what you need.** A group with one function is not lesser than a group with four.
- **The Paper Test.** Any software built to support this framework must only accelerate processes that could theoretically be done on paper.
